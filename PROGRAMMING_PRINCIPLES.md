# Принципи програмування в проєкті "Керування даними"

## 1. Single Responsibility Principle (Принцип єдиної відповідальності)
Кожна функція у проєкті виконує одну чітко визначену дію.
* **Опис:** Наприклад, функція `renderTable` відповідає виключно за оновлення DOM-елементів таблиці на основі даних, а не за логіку їх видалення чи додавання.
* **Код:** [Функція renderTable](index.html#L243-L257)

## 2. DRY (Don't Repeat Yourself)
Уникнення дублювання логіки шляхом винесення її в окремі функції.
* **Опис:** Функція `showAlert` використовується для відображення повідомлень у всіх частинах програми. Це дозволяє не писати логіку `style.display` та `setTimeout` кожного разу окремо.
* **Код:** [Функція showAlert](index.html#L137-L144)

## 3. Separation of Concerns (Розділення відповідальності)
Логіка інтерфейсу відділена від логіки обробки даних.
* **Опис:** Весь інтерфейс керується через зміну станів (класи `display: block/none`), а дані зберігаються в `localStorage`. Це дозволяє легко модифікувати зовнішній вигляд без зміни логіки збереження даних.
* **Код:** [Керування екранами в JS](index.html#L210-L215)

## 4. KISS (Keep It Simple, Stupid)
* **Опис:** Використання стандартних засобів браузера (`localStorage`, `JSON.stringify`) замість складних сторонніх бібліотек, що робить код легким для розуміння.
* **Код:** [Робота з даними](index.html#L112-L130)
